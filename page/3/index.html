Enter Bitcoin

By ipedrazas under BitcoinLondonstartups

Last month I decided that it was time for Bitcoin and I to spend more quality time together. So I went to MegaMine and opened an account. Why Megamine and not some others? well, truth is that they are local. It’s a Startup from London, you can go and visit them. Depending on the time of the day they will buy you a coffee or a beer and have a nice chat with you… and yes, their prices are super competitive!

I bumped into them because I was looking for a cheap datacenter to host my machines to build my OpenStack Empire, but that’s another story. Let’s focus!

If you want to get into Bitcoins there are two ways:

  • You can mine Bitcoins, aka Bitcoin mining
  • You can buy Bitcoins. In the same way that you would buy any other foreign currency.

The main difference is the price. Buying Bitcoins is like buying US Dollars, or British Pounds. (Buying and selling prices are different.) When mining you pay for the usage of a mining machine (or a pool), regardless of the price of the Bitcoin (well, that’s the theory). This means that under certain circumstances, buying Bitcoin is more convenient that mining.

buying Bitcoin is not as straightforward as many people think. Yes, there are places where you can go and buy Bitcoins. for example, this website. Now, if you notice, the website does not sell Bitcoins, the website will allow you to buy Bitcoins (from someone else), and because of that, every transaction has different conditions (looks dodgy, doesn’t it?)

At the beginning of the month SatoshiPoint presented their machines and I rushed to go and compare what I could get from them compared to what I could get from mining.

The difference was that the machines will give you your bitcoin straightaway (in reality it the transaction took around 13 minutes to go through) compared to the 5~ish days that I had to spend mining to get the same amount of Bitcoin.

As I see it, if you want your Bitcoins now, go and buy them out of some place. Make sure the website is the one selling the Bitcoins and not-yet-another-middle-man-in-the-middle.

If you want to squeeze your money and get a higher return… go and mine!

I’m not sure if Megamine will sell Bitcoins in the future or not, but I will be asking them next time I see them for a coffee or a beer.

 

Happy Mining!

Thoughts on Docker

By ipedrazas under Docker

 

Docker is an open-source project to easily create lightweight, portable, self-sufficient containers from any application

Vagrant

It’s not a secret that Vagrant changed my life as a developer. I’m sure that if you know me, you will have heard me talking about how great vagrant is.

Vagrant, in case that you don’t know, it’s a little tool that allows you to develop locally an execute your code inside a Virtual Machine. This, of course, gives you the advantage of not needing to setup your own machine (a painful activity that we all have been suffering in silence for years). In a nutshell, vagrant helps you to decouple your development environment from your workstation.

After Vagrant

Now, we cannot say that there’s a new guy in the block, because Docker has been around for a while (ehem…) but we do can say that Docker is not only going to change my life but it’s about to change everyone’s life.

Docker will change the way we run and distribute applications.

I don’t think I’m the right person to explain what Docker is. I can tell you how it has changed the way I work, or to put it differently, to change the things I do, but if you really want to understand how Docker works, it’s better you look at the resources at the bottom of the page.

Enter Docker

Docker is a linux container that allows you to run an application with a simple “RUN” command. Yes, there are a few other parameter but the idea is that you can launch an app (that it’s self contained) with a single command.

If you haven’t suffered the enlighting experience of trying to run a web application like WordPress, for example, you don’t know how complex things can turn very quickly. The ammount of software and things that have to be “in place” and configured properly is painful. How Docker can help you there? well, someone, for example me, will spend some time building a Docker image so people like you can just run it with a simple command.

Docker will lower the sys admin barrier. People who doesn’t know much about how to set up a web server suddenly can run their own wordpress instance (which is the idea behind of all those VPS with “one-click install”).

But there’s more, with Docker creating a cluster it’s easy. Ok, not trivial, but very feasible. Do you remember those complicated architectures you have seen in books? now you can build them in your laptop. I agree, a cluster under work load behaves very different than a cluster you could run in your workstation, but let’s face it, the fact that you can use docker to build a cluster of machines will help to speed up the future development of applications and solutions.

So, what I have been doing with Docker that has changed the way I see things?

well, I’m writing a little tool to manage bookmarks. The App has two provisioning tools. Vagrant (of course) and ansible scripts to deploy and configure the app in a VPS (like Digital Ocean, for example).

.dotMarks

The whole idea of this app is to self-host your bookmarks. Yes, I know, nobody seems to use bookmarks nowadays but that’s a different story. The idea is that you run it in your machine… and because of that you can see how Docker can make my life much easier: It’s like giving people an executable that they can run. Everything is self-contained and pre-configured. It’s perfect, it’s Awesome!

What about me?

By default I run 3 docker containers in my machine. One with the Dockerui (A web interface for Docker), One with ElasticSearch and one with MongoDB. I still use Vagrant to develop and run .dotMarks, but once it’s ready, I can tell you, it will run in its own docker container.

Marvel plugin for ElaasticSearch showing the state of my system

Where do we go from here

Good question. As I see it, there are three areas that are not under the Docker Radar but I think that once the right people understand the potential it will go through as a freight train:

  • Education: From the newbies that they need a (safe) playground to land, to the more experienced guy who wants to understand complex architectures, Docker has a huge potential in (self) education.
  • Application Distribution: WebApps are awesome, but distributing them is quite a pain… in fact, not many people do that because let’s face it, it’s hard. It’s not only shipping the code, it’s configuring the servers, and running the whole lot and… it’s complicated. Docker will bring a standard way of distributing applications. From wordpress to the latest trendy idea someone pushes to Github… I’m sure developers will start including Dockerfiles to their repos once they understand the wonders of linux containers
  • Support and Quality Control: Suddenly, testing applications in the real architecture locally is suddenly possible. Do you have customers with your system installed in-house. Replicating their environtment is a matter of linking images in a Dockerfile.

But don’t get too carried away, Docker has its amaizing Use Cases but it’s not for everyone. Yes, developers and SysAdmins love it… time will say if it will become mainstream.

Distributed Access

By ipedrazas under Londonstartups

Sharing information is not easy. We are animals of customs. Moving from one way of doing things to another takes time and it creates friction and it happens, over time that we accept the new way of doing things as the “way we do this”.

Working with a distributed team is strange. my team is not exactly distributed, but asynchronous. Working with people whoe are not next to you creates a barrier, but opens up tons of possibilities.

My wife recently asked  advise about how to deal with a distributed and/or asynchronous team. Technically we have so many options that it’s difficult to know which is the right choice.

These are the applications that in my experience, make a distributed async team to be as productive as working together elbow to elbow:

  • Serialised Chat: a chat where people can see the last 4 days of history. Chats are not only useful to “chat” but to keep track of what was going on.
  • Internal Blogs: This is the core of a team. Every single team should have an internal blog where they dump what’s happening. I know, it seems the same than the chat, but it’s not. Chats are prone to off topics and and are terriffic to let people know. Blogs allow transmitting info and gathering comments dead easy.
  • Skype/hangouts: not much to say here really. There are times that a simple phone call can sort out many problems.
  • Wiki: nothing beats the good old wiki to create a solid knowledge base.
  • Github: git it’s not only great for code, but for anything you need to pull-request
  • Dropbox: sharing made easy.

I do have tried things like basecamp, yammer or jive and they are great but they do not fit very well with small teams.

Vagrant: Up and Running By Mitchell Hashimoto

By ipedrazas under books

I wanted a book to get me up to speed with Vagrant and that’s exactly what Vagrant Up and Running does.

I have talked about Vagrant already in this blog (here and here) but if you have looked at Vagrant and you’re not using it yet because you “didn’t get it” let me tell you how I did it.

Books are great for building this baseline of knowledge. They have the advantage of curated structure and the disadvantage of getting outdated… but since my memory is quite terrible, what I want from a book is a better understanding of the technology and not how to type that command.

For the accurate sintaxis and instructions I fall back to google What I do is the following. When I’m going to learn something new I always start a new project where I can use that technology.

My projects are structured as follows:

  • prod
  • lab
  • test

So, depending what I’m going to do I add an entry to test (if it’s a quick test that can be deleted at any time) or to lab if it’s a project I want to use to learn a new technology. There’s nothing like building to get the learning going

For example, I’m building Cryptadoo as the way of learning Django, or with Grails I created my “Rewards” project.

Vagrant is different because you cannot build something, but it helps you to develop better. Still, I created a project with AngularJS to test Vagrant (ServerMarks).

For me the book provides a solid foundation for day to day use of Vagrant, with plenty of guidance for getting into more advanced usage. Automation is encouraged by Vagrant’s workflows, I would say even more: Automation is encourage in general.

New users and people with zero Vagrant experience will be able to pick up this book and learn everything they need to start using Vagrant and getting the most out of it.

If you’re like me, this book will help you loads because it illustrates the best use cases of Vagrant. I agree, the fact that V1.0 and V1.1 or V2.0 have some incompatibilities will force you to go out and look for the right command, but as I said before, books can get outdated pretty quickly, but there’s something that books give you than none of your random search will do: structure… and hey, you can read them without Internet connection for example: while commuting

If you want a step by step tutorial this Vagrant Up and Running will help you little, but as a way to accelerate learning it’s a wonderful book!

The Challenge

By ipedrazas under childrensecurity

Recently me and my wife have been having some problems with Internet and Children. I know, there are hundreds of filters and ways of preventing children to access content that it not appropriate to their age… and none seems to work well.

You can enable Safe Search in Google and YouTube Safety Mode. Let me show you acouple of Image Searches by Mighty Google with the Safe Search On, I searched for “Human Rights“:

So, yes, almost all the images are “safe”… almost. Thing is, if you disable that safe search the results contain less blood.

Other options are using a proxy that filters out the dodgy content but not many people know how to install those things that have evolved into add-ons or extensions that filter out certain pages.

The issue with those solutions is that you rely on a company that gathers your browsing data (and your children’s browsing data) and all of them have long Terms and Conditions and what really worries me is that as a company, you need to make money. Since they give away their product, I’m afraid that the real product is the user’s data.

So, as parents we have some options. But it’s not what us as parents can do, but what our children are going to do. An example would be WoT, a great idea with a big problem: children don’t understand it. WoT

“crowdsources ratings from a global community of millions of users, who rate and comment on websites based on their personal experiences”.

Wicked! Now when I search I see that little green/red icon next to the results… Do you think that children understand that? me neither.

Don’t get me wrong, I think it is a great idea and it is the way to go. They have options to “Block sites that are unsuitable for children“. So, in many ways it is better than using Mighty Google and its Safe Search.

 

Unfortunately WoT is not only focused on Children safety which is good and bad… but it is a great idea and I will be looking at them (and their API) more closely.

One of the best Add-ons for Firefox or Chrome is Web Filter by CloudAcl. It allows you Black List or White List websites. A parents/kids mode and it’s password protected… and they have a mobile version for your iPhone/iPad or your Android device.

Another option is to use a “Kid’s Safe Browser” like KidZui. Again, this is a great tool with the only issue of why do they need to gather and keep browsing history in their servers and not keep it in MY BROWSER!!! (the fact that they do that is a big red flag, since technically there’s no reason for taking my data somewhere else and by doing that your message is “I do it because that’s how I make money”. Again, me and my children are the product not the customer.

As it seems, the options out there are not as good as we might expect. Or they are not as efficient as we would like to or they are not specialised in children’s browing or they have a dodgy business model.

What I would like is to have an open source solution that stops my children from seeing things that are not appropriate for them. I’d like to have also a tool that allows me easily to monitor what is going on when I’m not there browsing with them.

I’m not sure if there’s such thing, what I know is that I haven’t found it yet…  Since I like technology and specially Raspberry Pis, I’m building a plug and play device that I call SafeBerry Pi and it’s basically the monitoring tool. Still very early stages but worry not, I will be writing about it as soon as I have something (stable) working, so, stay tunned!!

Monitoring Cron Jobs with Jenkins

By ipedrazas under DevOps

“How do you monitor your cron jobs?”

“We just don’t…”

Monitoring is one of those things that yo should do: monitor EVERYTHING… or at least as much as you can.

One of the most slippery cases are our cron jobs. Many of them don’t report anything, many of them run without anyone knowing they’re there… And even worse, if they fail, they fail silently: as this cron job I disabled because it had been failing since 2008 (only 5 years or every day failures).

That was the last straw, after removing that entry from the crontab I went to my boss and we decided to tackle the problem. This client uses Nimbus but it seems that the Nimbus guy was not very happy on having loads of cron jobs added so I suggested using Jenkins.

In this case we decided to use Free Style Software Projects since they allow us to run the cron job manually

JENKINS_HOME=http://myserver.acme.org/path/to/jenkins/
0 * * * *     export JENKINS_HOME=$JENKINS_HOME; java -jar jenkins-core-*.jar "backup" backup.sh 2>&1 > /dev/null

You can find more info in the Jenkins Wiki but if you want to know more or have any doubts, just ping me on Twitter
